# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Что это за репозиторий

Два взаимосвязанных слоя:

1. **Excel-система управления портфелем** — шаблоны и регламенты для ООО «Меридиан Инжиниринг» (`00_Регламенты/`, `02_Шаблоны/`, `03_Проекты/`)
2. **Python-приложение `tracker_mvp/`** — локальная operational-система для проектов АСУ ТП, которая заменяет Excel-карточку проекта как источник истины

Всё содержимое — документы, имена полей, комментарии в коде, русскоязычный UI — на **русском языке**.

## Приложение tracker_mvp

### Стек

| Слой | Технология |
|------|-----------|
| Backend | FastAPI |
| UI | Jinja2 templates + CSS (без React/TypeScript) |
| База данных | SQLite (per-project workspace) |
| ORM | SQLAlchemy 2.x (mapped_column) |
| Миграции | ADD COLUMN через `PRAGMA table_info` в `db.py` |
| Тесты | pytest + httpx |

### Запуск

```powershell
# из папки tracker_mvp
..\.venv\Scripts\python.exe -m pip install -r requirements.txt
..\.venv\Scripts\python.exe -m uvicorn app.main:app --reload
```

Приложение доступно по `http://127.0.0.1:8000`. Проверка: `GET /health`.

### Запуск тестов

```powershell
# из папки tracker_mvp
..\.venv\Scripts\python.exe -m pytest tests/
# один файл
..\.venv\Scripts\python.exe -m pytest tests/test_phases.py -v
```

### Архитектура приложения

```
tracker_mvp/
  app/
    main.py            ← точка входа FastAPI, монтирует все роутеры
    db.py              ← SQLAlchemy engine, get_db(), init_db(), ADD COLUMN миграции
    models.py          ← все SQLAlchemy-модели
    workspace.py       ← управление active workspace (app_state.json → tracker.sqlite)
    templates.py       ← Jinja2 environment
    routes/            ← по одному файлу на каждый раздел UI
    services/          ← бизнес-логика (event_log, p3_service, phase_service, ...)
    templates/         ← Jinja2-шаблоны (.html)
    static/styles.css
  tests/               ← pytest, каждый файл тестирует один модуль
  data/
    app_state.json     ← сохранённый путь к активному workspace
  requirements.txt
```

### Project workspace

Каждый проект — отдельная папка на диске:

```
ProjectFolder/
  project_config.json   ← манифест (project_id, пути к папкам)
  project_context.md    ← человекочитаемый контекст для агентов
  agent_rules.md        ← разрешения для ИИ-агентов
  tracker.sqlite        ← вся база данных проекта
  documents/
  archive/
  backup/
  exports/
  agents/inbox/ work/ outbox/ logs/
```

Активный workspace хранится в `tracker_mvp/data/app_state.json`. При переключении workspace engine SQLAlchemy пересоздаётся (`db.py::get_engine()`).

### Модель данных (models.py)

Все PK — UUID-строки. Числовые поля (суммы, проценты) хранятся как `String`, нормализуются при импорте Excel.

Ключевые таблицы:

| Таблица | Назначение |
|---------|-----------|
| `projects` | Паспорт проекта |
| `tasks` | Задачи (статусы: Бэклог / В работе / Тестируется / Готово / Заблокировано) |
| `documents` | Реестр документов с локальным путём |
| `work_items` | Строки работ, импортированные из расчёта КП/СС |
| `procurement_items` | Закупочные позиции с группировкой и статусами |
| `risks` | Реестр рисков с risk_score и risk_level |
| `p3_activities` | P3.express чек-лист (governance layer РП) |
| `p3_cycles` | Циклы P3.express |
| `project_phases` | Фазы инженерного жизненного цикла АСУ ТП |
| `technical_audits` | Техаудиты фаз с решением по переходу |
| `entity_links` | Универсальные связи source→target между любыми сущностями |
| `agent_proposals` | Предложения ИИ-агентов, ожидающие решения человека |
| `company_people` | Справочник сотрудников для выбора ответственных |
| `events` | Журнал всех значимых действий (actor, before/after state) |
| `settings` | key/value настройки приложения |

### Миграции

Нет Alembic. Новые колонки добавляются через `ADD COLUMN ... DEFAULT ''` в `db.py`. Каждая группа колонок — отдельная функция `migrate_*_columns()`, которая проверяет `PRAGMA table_info` перед добавлением.

**При добавлении новой колонки**: добавить её в `models.py` и добавить `ALTER TABLE ... ADD COLUMN` в соответствующую `migrate_*` функцию.

### Правила разработки

Из `00_Регламенты/MVP_трекер_АСУТП/11_Правила_разработки.md`:

- Комментарии обязательны для: нового маршрута или сервиса, неочевидной бизнес-логики, импорта Excel, операций с файловой системой, мест с временным MVP-решением.
- Комментарий объясняет **зачем** (связь с процессом проекта), не **что**.
- Любое новое изменение должно затрагивать цепочку: `UI → model → route/service → event → test`.

### Агентская логика

Агент формирует предложения в `agent_proposals` — человек принимает или отклоняет. Агент не меняет данные напрямую. Результаты агента сохраняются в `agents/outbox/` внутри workspace.

## Excel-система управления портфелем

Два взаимосвязанных файла:

```
Портфель проектов.xlsx
  └── "Активные"   — ссылки на карточки проектов через внешние формулы
  └── "Завершённые" — архив (вставка значениями)
  └── "Сводка"     — сводный дашборд

Шаблон_Карточка проекта.xlsx  (02_Шаблоны/Шаблон проекта.xlsx)
  └── "Проект"    — паспорт: клиент, стадия, команда, сроки, %
  └── "Задачи"    — задачи проекта
  └── "Закупки"   — реестр оборудования
  └── "Бюджет"    — бюджетные категории
  └── "Риски"     — реестр рисков
```

Данные читаются формулами вида:
```
='I:\_Проекты\[Заказчик]\[Объект]\00_Управление\[Карточка проекта.xlsx]Проект'!C6
```

Обновление портфеля: **Ctrl+Alt+F9**.

### Ключевые поля листа «Проект»

| Ячейка | Поле |
|--------|------|
| C3 | Заказчик |
| C4 | Объект / название проекта |
| C5 | Номер договора |
| C6 | Статус: В работе / Приостановлен / Завершён / Отменён |
| C7 | Текущая стадия (КТ-0 … КТ-5) |
| C8–C9 | Плановые даты начала и окончания |
| C10, C13 | Фактические даты начала и окончания |
| C11–C12 | РП и ведущий инженер |
| C15 | % выполнения |

### Стадии проекта (КТ)

| Стадия | Условие перехода |
|--------|-----------------|
| КТ-0 Инициация | Карточка создана, команда назначена, план утверждён |
| КТ-1 ТЗ | ТЗ подписано заказчиком |
| КТ-2 Проектирование | Проектная документация выпущена |
| КТ-3 Программирование | Код ПО готов |
| КТ-4 FAT | Заводские испытания пройдены |
| КТ-5 ПНР | Объектные испытания пройдены, проект закрыт |
