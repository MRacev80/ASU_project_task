# MVP-STATUS: статус реализации

## Назначение

Документ фиксирует, что уже реализовано в коде MVP, что находится в частично готовом состоянии, а что остается следующим этапом. Его цель - не возвращаться к уже принятым и выполненным решениям без отдельной причины.

Актуально на момент фиксации: 25.05.2026.

## Общий статус

Первая техническая итерация MVP реализована частично, но уже есть рабочий локальный skeleton приложения.

Готово:

- локальное FastAPI-приложение;
- серверный UI на Jinja2 templates;
- SQLite;
- создание и открытие папки проекта;
- создание структуры папки проекта;
- база данных внутри папки проекта;
- базовая карточка проекта;
- задачи;
- документы;
- журнал событий;
- вкладки-листы карточки проекта;
- фундамент для агентской рабочей области.

Не готово:

- полноценные разделы `Бюджет`, `Закупки`, `Риски`, `P3.express`;
- импорт из Excel-шаблона;
- агентские предложения;
- интеграция с LLM API;
- бэкап и архив через UI;
- экспорт.

## Что считать завершенным и не открывать заново

Следующие решения считаются закрытыми для текущего MVP:

- приложение локальное;
- стек первой итерации: FastAPI + Jinja2 + HTMX-compatible routes + SQLite;
- данные проекта хранятся в папке проекта;
- Excel-шаблон является источником структуры, но не источником истины;
- разделы проекта должны быть отдельными вкладками-листами;
- агент работает только внутри структуры папки проекта;
- агент не меняет базу и документы напрямую.
