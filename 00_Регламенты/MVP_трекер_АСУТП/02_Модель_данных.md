# MVP-DATA: модель данных

## Назначение

Документ описывает модель данных MVP для локальной реализации на SQLite. Модель должна быть простой, но расширяемой: проекты, задачи, документы, агенты, предложения агентов, события и настройки.

## Основные таблицы

- `projects`;
- `tasks`;
- `documents`;
- `task_documents`;
- `agents`;
- `agent_proposals`;
- `events`;
- `settings`.

## Таблица `settings`

| Поле | Тип | Назначение |
|---|---|---|
| `key` | TEXT | Ключ настройки |
| `value` | TEXT / JSON | Значение настройки |
| `updated_at` | DATETIME | Дата обновления |

Примеры настроек:

- путь к рабочей папке системы;
- путь к папке резервных копий;
- путь к архиву документов;
- параметры подключения к LLM API;
- выбранный агент по умолчанию.

## Основные связи

- один проект содержит много задач;
- один проект содержит много документов;
- одна задача может быть связана с несколькими документами;
- один документ может быть источником нескольких задач;
- один агент может создать много предложений;
- одно предложение агента может быть преобразовано в одну задачу;
- все значимые действия фиксируются в журнале событий.

## Правила целостности

- задача всегда принадлежит проекту;
- документ всегда принадлежит проекту;
- предложение агента всегда принадлежит проекту и агенту;
- задача, созданная из предложения агента, хранит ссылку на это предложение;
- удаление проекта, задачи или документа в MVP лучше заменить архивированием или сменой статуса;
- все действия агента создают запись в `events`.
