# АСУТП Tracker MVP

Локальный MVP трекера задач проекта АСУТП.

Стек первой итерации:

- FastAPI;
- Jinja2 templates;
- HTMX-compatible routes;
- SQLite;
- SQLAlchemy;
- простой CSS.

## Что реализовано

- локальный запуск на `localhost`;
- создание и открытие папки проекта;
- создание структуры папки проекта: `project_config.json`, `tracker.sqlite`, `documents/`, `archive/`, `backup/`, `agents/`;
- создание проектов;
- список проектов;
- создание задач;
- изменение статуса задач;
- добавление документов как записей с локальным путем;
- журнал событий;
- базовые настройки путей.

## Установка

Из папки `tracker_mvp`:

```powershell
..\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

## Запуск

Из папки `tracker_mvp`:

```powershell
..\.venv\Scripts\python.exe -m uvicorn app.main:app --reload
```

После запуска открыть:

```text
http://127.0.0.1:8000
```

Проверка backend:

```text
http://127.0.0.1:8000/health
```

## Данные

Приложение работает с папкой проекта. При создании папки проекта формируется структура:

```text
ProjectFolder/
  project_config.json
  project_context.md
  agent_rules.md
  tracker.sqlite
  documents/
  archive/
  backup/
  agents/
    inbox/
    work/
    outbox/
    logs/
  exports/
```

Активная папка проекта хранится в служебном файле:

```text
tracker_mvp/data/app_state.json
```

Если папка проекта еще не выбрана, приложение показывает экран создания или открытия папки.

## Проверочный сценарий

1. Запустить приложение.
2. Открыть `http://127.0.0.1:8000`.
3. Создать или открыть папку проекта.
4. Создать проект в выбранной папке.
5. Создать задачу.
6. Перевести задачу в статус `В работе`.
7. Добавить документ.
8. Проверить журнал событий.
9. Перезапустить приложение.
10. Убедиться, что данные сохранились в `ProjectFolder/tracker.sqlite`.

## Ограничения первой итерации

- нет агентского анализа документов;
- нет очереди предложений агента;
- нет бэкапа и архива документов;
- нет авторизации;
- нет экспорта;
- нет многопользовательской работы.
