# Автотесты MVP

Тесты проверяют ключевые сценарии из документа `00_Регламенты/MVP_трекер_АСУТП/12_Тестовые_сценарии_и_юзкейсы.md`.

## Запуск

Из папки `tracker_mvp`:

```powershell
..\.venv\Scripts\python.exe -m pytest -q
```

## Что покрыто

* создание и открытие project workspace;
* структура папки проекта;
* импорт расчетов КП/СС на уровне парсера;
* фильтрация нулевых и пустых строк бюджета;
* группы закупок и округление чисел;
* ручные закупки: создание, редактирование, удаление;
* запрет удаления импортированной закупки ручным маршрутом;
* допустимые статусы закупок;
* задачи: создание и смена статуса;
* риски: создание, редактирование, удаление;
* бюджетные фильтры и итоги;
* группировка бюджета по разделам и итоги по разделам;
* фильтры закупок;
* фильтры рисков и расчет уровня риска;
* универсальные связи сущностей, человекочитаемые названия и комбинированный выбор объекта связи;
* graph-view связей и фильтры graph-view;
* before/after-снимки в журнале;
* фильтры журнала по событию, сущности и источнику;
* безопасная ошибка импорта несуществующего расчета;
* P3.express: расчет текущей фазы;
* P3.express: циклы, дата начала, дата окончания и длительность;
* P3.express: подсветка просроченных действий и циклов;
* P3.express: обновление действий с before/after;
* agent proposals: создание, принятие, отклонение и применение принятого предложения;
* справочник людей компании и скрытие неактивных сотрудников;
* фазы проекта: автосоздание 11 фаз, редактирование, решение по переходу, техаудит и состояние проекта;
* `/health` через FastAPI TestClient.

## Важное

Тесты используют временный workspace через `tmp_path` и не должны трогать рабочий `app_state.json` или реальную базу проекта.
