# Правила разработки

## Комментарии в коде

- В новом коде оставлять короткие комментарии к структуре модуля, назначению сервисов, маршрутов, моделей и сложных блоков логики.
- Комментарий должен отвечать на вопрос "зачем это сделано" или "как устроен поток данных", а не пересказывать очевидную строку кода.
- Для импортов из Excel, миграций БД, работы с папкой проекта, агентами и файловой системой комментарии обязательны.
- Если добавляется новый файл, в начале файла должна быть понятна его роль: через название, структуру кода или короткий комментарий рядом с главным классом/функцией.
- При изменении существующего кода добавлять комментарии там, где без них пользователю будет сложно понять дальнейшую поддержку.

Пример хорошего комментария:

```python
# Импорт расчета пересоздает производные строки, потому что источник истины - Excel-файл.
```

Пример лишнего комментария:

```python
# Увеличиваем счетчик на 1.
```
